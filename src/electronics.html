Webcam Controller

My first time working with serial communication beyond Arduino interfaces and directly interacting with a computer. This webcam control system enables zoom adjustment and reorientation using an Arduino controller, along with an image capture feature that saves photos directly to the computer. It was cool to see how the Arduino could directly manipulate the computer interface and control the webcam in real-time. I also learned about UART and serial data transmission where the Arduino and computer could communicate with each other as long as they used the same baud rate without needing a clock signal for synchronization.

Features: Serial Communication (UART), Tilt Sensor, Java (Processing), Arduino, C/C++

Temperature Regulation Controller

I wanted to simulate a temperature-sensitive fan system that could be manually adjusted to simulate heating or cooling conditions. Even though there is no physical fan connected in the demo, the project was a way for me to test out DC motor control using an H-bridge. The LCD would display the current temperature and fan speed, while the fan automatically changes its speed based on the temperature. The device also included an automatic shutdown feature to simulate overheating conditions. I implemented the Arduino's EEPROM library to restore its last cooling/heating mode after reset or power loss. This taught me about electrically erasable programmable read-only memory (EEPROM) where the microcontroller can store small amounts of data in non-volatile memory. It was also the first time I used an external power source (9V battery) to power the motor where I learned proper circuit design and power handling. Before the H-bridge, I learned how to use a diode to prevent back EMF generated by the motor from damaging the circuit.

Features: Temperature Sensor, DC Motor, H-Bridge, EEPROM, Arduino, C/C++, LCD Display

Bomb Defusal Simulation

This project gave me the chance to combine everything I had learned so far about input/output systems and I wanted to design an interactive game that had a time pressure element. The servo motor was used to simulate a lock mechanism that would produce the correct four digit code to disarm the bomb. In the first iteration, the servo motor drew a lot of current, causing the LCD to appear dim and unreadable. I added a capacitor to store local charge, which smoothed out voltage dips and prevented brownouts.

Features: Time-Sensitive Design, Servo Motor, Capacitor, Arduino, C/C++, LCD Display

Dual-Mode Piano Trainer

I wanted to create an interactive system that incorporated UI/UX elements and to practice different piezo buzzer tones. The dual-mode piano trainer features a menu selection where you can pick between a free-play mode or a practice mode. The free-play mode is just a simple five keyboard piano setup that plays different tones based on the key pressed. The practice mode provides three songs to choose from: Hot Cross Buns, Mary Had a Little Lamb, and Ode to Joy. Once selected, the device takes you through the song step-by-step with visual cues and audio error feedback. I learned how to make the LCD scroll through on each successful key press, along with an auto-scroller for longer texts.

Features: UI/UX Design, Mode-Switching Control, Piezo Buzzer, Arduino, C/C++, LCD Display

Morse Code Simulator

This was my first big jump into embedded systems and I wanted to create a simple device that took in Morse code inputs and translated them into text. The LED and piezo buzzer produced visual and audio feedback for each button press. It was also my first time using an LCD screen and noticed the wire connection was different from the simulation I ran. This was my first experience learning about I2C and parallel communication protocols where the HD44780 LCD ran on parallel which meant more wires to connect. In I2C protocol there are two signal lines, SCL and SDA which allows for more efficient communication with fewer wires. I also learned about master/slave architecture and how data is transmitted in packets using the SDA while the SCL helps synchronize transfer. The setup also resulted in a dim display which meant I had to apply a source for the backlight pins on the LCD without blowing it up.

Features: I2C Communication, Parallel Communication, Piezo Buzzer, Arduino, C/C++, LCD Display

Nightlight Detector

I wanted to experiment with different sensors and designed a nightlight detector that senses ambient light levels while changing the RGB LED white light intensity. It was my first time working with an RGB LED and I learned how to mix colors based on the analog reading from the phototransistor. I had to map and constrain the analog reading to a range of 0-255 to get the correct colour intensity. For analog sensors, there was some calibration done to get the correct range of values to work with.

Features: Phototransistor, RGB LED, Arduino, C/C++

Whack-A-Mole Game

This was my very first physical embedded systems project where I took a simulated assignment from class and built it in real life. The game logic involved random LED activations which required the player to hit the correct button within the time frame. There is also an inactivity timer that resets the game and flashing LED sequences based on performance. I learned about button debouncing and non-blocking timing using the millis() function to keep track of time without pausing the program.

Features: Real-Time Game Logic, Button Debouncing, Non-Blocking Timing, Arduino, C/C++
