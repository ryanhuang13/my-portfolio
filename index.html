Recent Projects Highlight

Live Weather Monitor

I designed a weather monitoring system that accesses the weather and temperature from Vancouver, Toronto, Ottawa, and Markham. I learned how to use the OpenWeatherMap REST API to fetch live weather data from the internet and display it on an LCD screen. The system includes an RGB LED that changes colour based on the weather conditions and audio alerts for temperature changes. This was also the first time where I had serial communication between the Java program and the Arduino, where both sides were writing on the same serial port to send and receive data. Normally, I would only have the Arduino sending data to the computer, but this time the computer sends data to the Arduino to change the LED or buzzer sound. I also accessed the internet from my program for the first time setting up URL connection and parsing JSON data, extending beyond local data processing.

Features: REST API Integration, JSON, Serial Communication (UART), Piezo Buzzer, RGB LED, Java (Processing), Arduino, C/C++

4-Bit Arithmetic Logic Unit (ALU) Design & Verification

For my first Verilog project, I designed a 4-bit Arithmetic Logic Unit (ALU) supporting addition, subtraction, AND, and OR operations. The design includes carry, zero, negative, and overflow flags to indicate result conditions. To verify functionality, I created a self-checking testbench that automatically compared ALU outputs against expected values, helping me practice test case handling and QA procedures. I also used GTKWave waveform analysis to debug signal propagation over time and to inspect how flags behaved during edge cases. Running simulations with Icarus Verilog (.vvp producing .vcd for GTKWave) gave me practical experience with the HDL toolchain. Through this project, I gained hands-on understanding of Verilog syntax, binary addition/subtraction (including wrapping and carry/borrow logic), and the role of MSB/LSB in signed 4-bit arithmetic.

Features: Verilog, GTKWave, Logic Analyzer

Temperature Regulation Controller

I wanted to simulate a temperature-sensitive fan system that could be manually adjusted to simulate heating or cooling conditions. Even though there is no physical fan connected in the demo, the project was a way for me to test out DC motor control using an H-bridge. The LCD would display the current temperature and fan speed, while the fan automatically changes its speed based on the temperature. The device also included an automatic shutdown feature to simulate overheating conditions. I implemented the Arduino's EEPROM library to restore its last cooling/heating mode after reset or power loss. This taught me about electrically erasable programmable read-only memory (EEPROM) where the microcontroller can store small amounts of data in non-volatile memory. It was also the first time I used an external power source (9V battery) to power the motor where I learned proper circuit design and power handling. Before the H-bridge, I learned how to use a diode to prevent back EMF generated by the motor from damaging the circuit.

Features: Temperature Sensor, DC Motor, H-Bridge, EEPROM, Arduino, C/C++, LCD Display
