Recent Project Highlights

Automated Sensor Logger & Reporter

My goal for this project was to automate sensor acquisition from an Arduino and interface it with Linux for QA workflows. Since many embedded systems integrate with Linux environments, I wanted hands-on experience with shell scripting for automated testing. I developed an automation pipeline that collected sensor data, logged it into CSV with timestamps, generated plots, flagged anomalies, and emailed reports with logs and visualizations. This was my first time working in Linux using WSL and Ubuntu. I learned to transfer files from Windows into Linux, configure Gmail integration by editing Linux mutt settings with nano, and automate workflows with Bash. The main challenge was serial communication due to WSL limitations where it doesn't support direct access to hardware COM ports, so I installed com2tcp to forward Arduino data into the Linux environment by hosting a local TCP bridge directly on my computer. I debugged issues with baud-rate mismatches, COM permissions, IP addressing, and bypassing DSR handshakes in the TCP protocol. By comparing Arduino's serial monitor with Linux netcat (nc) outputs, I was able to validate and properly parse the incoming data stream.

Features: Linux OS, Bash, TCP/IP Protocol, Arduino, C/C++, Temperature Sensor

Spotify Controller

For this project I designed a Spotify controller that can play/pause, skip forward/back, and adjust the volume. I integrated an LCD display to show the current song, artist, remaining time, and added auto-scrolling for longer titles. I learned how to handle token-based authentication (OAuth) to access my personal Spotify account, and leveraged the pyserial and spotipy packages to interact with playback, volume, and track information. It was also interesting to explore serial communication between Python and Arduino, encoding Python's Unicode strings into bytes for transmission and decoding bytes received from the Arduino. Since this is my third project working with API requests, I am a lot more comfortable parsing JSON data and receiving/transmitting data over serial to the Arduino. I really liked this project as it gave me the opportunity to explore managing continuous serial data streams, handling redirect URIs for OAuth authentication, and integrating hardware with the Spotify Web API to create a responsive, interactive music controller.

Features: Python, Spotify Web API, Token-Based Authentication (OAuth), Playback Control, Serial Communication, Arduino, C/C++, LCD Display

4-Bit Arithmetic Logic Unit (ALU)

For my first Verilog project, I designed a 4-bit Arithmetic Logic Unit (ALU) supporting addition, subtraction, AND, and OR operations. The design includes carry, zero, negative, and overflow flags to indicate result conditions. To verify functionality, I created a self-checking testbench that automatically compared ALU outputs against expected values, helping me practice test case handling and QA procedures. I also used GTKWave waveform analysis to debug signal propagation over time and to inspect how flags behaved during edge cases. Running simulations with Icarus Verilog (.vvp producing .vcd for GTKWave) gave me practical experience with the HDL toolchain. Through this project, I gained hands-on understanding of Verilog syntax, binary addition/subtraction (including wrapping and carry/borrow logic), and the role of MSB/LSB in signed 4-bit arithmetic.

Features: Verilog, GTKWave, Logic Analyzer
