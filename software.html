Automated Sensor Logger & Reporter

My goal for this project was to automate sensor acquisition from an Arduino and interface it with Linux for QA workflows. Since many embedded systems integrate with Linux environments, I wanted hands-on experience with shell scripting for automated testing. I developed an automation pipeline that collected sensor data, logged it into CSV with timestamps, generated plots, flagged anomalies, and emailed reports with logs and visualizations. This was my first time working in Linux using WSL and Ubuntu. I learned to transfer files from Windows into Linux, configure Gmail integration by editing Linux mutt settings with nano, and automate workflows with Bash. The main challenge was serial communication due to WSL limitations where it doesn't support direct access to hardware COM ports, so I installed com2tcp to forward Arduino data into the Linux environment by hosting a local TCP bridge directly on my computer. I debugged issues with baud-rate mismatches, COM permissions, IP addressing, and bypassing DSR handshakes in the TCP protocol. By comparing Arduino's serial monitor with Linux netcat (nc) outputs, I was able to validate and properly parse the incoming data stream.

Features: Linux OS, Bash, TCP/IP Protocol, Arduino, C/C++, Temperature Sensor

Live ESPN Scoreboard

I practiced with API request handling by developing an Arduino-based real-time scoreboard system that fetches live NBA, NFL, and MLB data from ESPN API endpoints, navigating nested JSON structures unique to each sport. Since I had to work with multiple JSON formats, I practiced proper, robust debugging techniques by using serial print statements to verify accurate data transfer and handle edge cases. The LCD dynamically displays team scores, game clocks, and start times, while LED indicators would flash when a team scores. I learned to use the Arduino/C++ struct to encapsulate reusable functionality for button debouncing and LED flashes, improving modularity and code efficiency. I ran into challenges with parsing through different API formats and getting the information I needed.

Features: REST API Integration, JSON, Serial Communication (UART), Java (Processing), Arduino, C/C++

Personal Project Website

This is my first website that I designed and built from scratch with no external templates. I learned how to use HTML to design the overall structure of my website and define key elements for organization. Then I used CSS to add colour and improve the visual presentation. Finally, I included some JavaScript scripts to add interactivity and responses to button clicks or mouse movements for a more engaging experience. This project also taught me a lot about user interface and user experience (UI/UX) design because I wanted to make sure that the most relevant pieces are highlighted. I aimed for a more minimalistic design, with darker and neutral tones to make it easier on the eyes. I also included different colour schemes for each project page to create a new theme for each skill area. The images are also enlargeable using a lightbox and modal concept in CSS and JavaScript to make it easier to see the photos. There are also clickable YouTube links for video demonstrations that automatically takes the user to a new page.

Features: HTML, CSS, JavaScript, UI/UX Design

Live Weather Monitor

I designed a weather monitoring system that accesses the weather and temperature from Vancouver, Toronto, Ottawa, and Markham. I learned how to use the OpenWeatherMap REST API to fetch live weather data from the internet and display it on an LCD screen. The system includes an RGB LED that changes colour based on the weather conditions and audio alerts for temperature changes. This was also the first time where I had serial communication between the Java program and the Arduino, where both sides were writing on the same serial port to send and receive data. Normally, I would only have the Arduino sending data to the computer, but this time the computer sends data to the Arduino to change the LED or buzzer sound. I also accessed the internet from my program for the first time setting up URL connection and parsing JSON data, extending beyond local data processing.

Features: REST API Integration, JSON, Serial Communication (UART), Piezo Buzzer, RGB LED, Java (Processing), Arduino, C/C++

Muscle Movement Data & Signal Processing

In this project I analyzed muscle movement using surface electromyography (EMG), motion tracking, and force plate data. I used a combination of Python and MATLAB to process and plot the data, including filtering, normalization, and statistical analysis. The purpose of this project was to assess muscle activation timing and force generation for grip strength. Another study was to analyze stroke rehabilitation by simulating impaired hand movements and quantifying kinematic movement patterns. The force plate study was done to measure center of pressure (CoP) and center of mass (CoM) for different balancing techniques. This was done to compare proprioception and balance control during rehabilitation exercises.

Features: Python, MATLAB, EMG, Motion Tracking, Biomechanics, Force Plate, Signal Processing, Data Analysis

Computational Modeling of Biological Systems

These are some plots from my computational modeling projects in biophysics. I worked with various models including gene circuits, predator-prey population dynamics, Michaelis-Menten and inhibition enzyme kinetics. I used Python to simulate and solve specific problems. For example, I simulated gene circuits using ordinary differential equations (ODEs) to model transcriptional regulation and feedback loops. I also performed stability analysis on population models to evaluate equilibrium behaviour and dynamics. I used linear and non-linear regression fitting to enzyme kinetics data to solve for system parameters. Finally, I solved boundary value problems (BVP) using the shooting method for mass transport and reaction-diffusion systems.

Features: Python, ODE Simulation, Kinetic Modeling, Data Fitting, Stability Analysis, Enzyme Kinetics, Shooting Method
